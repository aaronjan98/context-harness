# Session 2026-07-04

## Focus
TAA playlist automation — building out the autonomous ChatGPT bridge loop so CF can drive multi-step homelab tasks (Lidarr, Navidrome, qBittorrent) without manual intervention.

## What was built

### CF Bridge extension — permanent install
- Added `browser_specific_settings.gecko.id: "contextforge-bridge@local"` to manifest
- Signed via `web-ext sign --channel=unlisted` (AMO free signing)
- NixOS: moved Firefox from `environment.systemPackages` to `programs.firefox` with `ExtensionSettings` force-install policy pointing at the signed `.xpi`
- Key lesson: Firefox release requires Mozilla signing even for enterprise force-install; `file://` URLs alone are not enough

### CF Bridge extension — base URL auto-capture
- `ensureChatGPTTab` now matches root `https://chatgpt.com/` when stored URL has no path
- After injecting first message into a base-URL tab, `captureConversationUrl` polls the tab URL until it gains `/c/<id>` and saves it back

### Auto-run improvements
- **Parse failure auto-retry**: when `extractFirstToolCall` returns null but content has `contextforge-tool`, CF automatically sends "please re-issue using heredoc" instead of silently failing
- **No-command streak circuit breaker**: when last assistant message has no tool call, auto-send continuation prompt up to 2 times; on 3rd miss, send Pushbullet notification and stop
- **Continue button**: visible in toolbar when last message is assistant with no tool call and nothing running
- **Notification endpoint**: `POST /api/notify` calls Pushbullet for arbitrary alerts from frontend

### Command classification fixes
- SSH: classify by remote command (parse via shlex + string fallback for complex quoting), not by SSH itself
- `docker exec`: removed from CONFIRM list (transport layer like SSH)
- `>` redirect: `(?<!\S)>(?![>=])` — only flags redirects preceded by whitespace; excludes `2>/dev/null`, XML `</tag>`, `/tmp/` targets, `/dev/null` targets
- `apt-get` pattern: scoped to command-start `(^|[;&|` + "`" + `]\s*)` so `command -v apt-get` doesn't trigger it

### JSON sanitizer fix
- `sanitizeJsonNewlines` and `extractFirstToolCall` now re-escape any `\X` where X is not a valid JSON escape char (e.g. `\$` → `\\$`)
- Fixes `bash: bad escaped character` errors from shell variable escapes in tool calls

### System prompt additions
- `timeout_seconds: 3600` guidance for long-running polls
- Heredoc pattern for complex scripts (avoids JSON → shell → python -c → Python string nesting)
- "Don't pause for status updates — include next command in same response"
- "Work on parallel tasks while waiting for downloads; use polling loop command"

### Auto-scroll
- Added `useEffect` watching `toolStreamLog` to scroll virtualizer to last item as execution log grows

## Key debugging insight
`2>/dev/null` is `fd>file` not `cmd > file` — digit before `>` means file descriptor redirect, not overwrite redirect. The `(?<!\S)` lookbehind correctly distinguishes these by checking for whitespace before `>`.
