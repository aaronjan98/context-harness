## ChatGPT bookmarklet checkpoint
- User chose the bookmarklet approach for v1 instead of building a browser

## ChatGPT table export fix
- User tested the bookmarklet with a ChatGPT Portuguese-learning conversation
  and found that table formatting did not survive import into Context Forge.
- Root cause had two parts: the exporter used `innerText`, which flattened HTML
  tables, and the frontend Markdown renderer did not include GitHub-Flavored
  Markdown support for pipe tables.
- Added `remark-gfm` to the frontend renderer so imported pipe tables render as
  actual tables.
- Updated the readable ChatGPT DOM exporter and the bookmarklet to convert
  ChatGPT `<table>` elements into Markdown pipe tables before copying the
  transcript.
- Added basic table styling under `.cf-message-content`.
- Important manual step: any already-installed Firefox bookmarklet must be
  updated by replacing its URL with the new contents of
  `tools/chatgpt-dom-export.bookmarklet.js`.

## Multi-block assistant turns and attachment notes
- User found a recurring ChatGPT pattern where an assistant turn contains an
  initial message, a `Thought for ...` / tool section, and then a final answer.
  Context Forge only imported the first visible assistant block.
- Root cause: the exporter selected only the first
  `[data-message-author-role="assistant"]` root inside each ChatGPT turn.
- Updated the exporter to collect all matching message roots for the turn,
  serialize each content root, and join them into one canonical assistant
  message.
- Added lightweight visible-attachment export: file tiles with aria labels are
  represented as Markdown attachment notes such as `> [!attachment]` plus the
  visible file names. The exporter does not and should not attempt to scrape
  file contents out of ChatGPT.
- Regenerated `tools/chatgpt-dom-export.bookmarklet.js`; installed browser
  bookmarks must be updated again.

## Display math empty-attribute fallback fix
- User retested the custom GPT math conversation and display math still did not
  import. Plain browser copy showed the formulas as stacked visual text, while
  Context Forge showed empty gaps after lines like `Let`, `Then`, and `So`.
- Root cause likely included empty math attributes: the exporter used nullish
  coalescing for fallback fields, so an empty `aria-label=""` or `alttext=""`
  could stop extraction before trying MathML text or visible text.
- Updated math extraction to use the first non-empty fallback value instead of
  the first non-null value.
- Broadened math detection beyond KaTeX classes to include raw `<math>`,
  `<mjx-container>`, `role="math"`, and `data-latex` / `data-tex` nodes.
- This should preserve at least visible fallback content for display math when
  original TeX is unavailable. It is not a full MathML-to-TeX converter.
- Regenerated `tools/chatgpt-dom-export.bookmarklet.js`; installed browser
  bookmarks must be updated again.

## Top-level KaTeX display block export fix
- User provided the actual ChatGPT HTML for the missing counterexample math.
- The real root cause was that ChatGPT emits display math as top-level
  `<span class="katex-display">...</span>` siblings inside the `.markdown`
  container. The exporter block serializer only kept known block tags such as
  `p`, `div`, `h2`, `table`, and `ul`, so these top-level span display blocks
  were filtered out before math extraction ran.
- Updated `blockChildrenToMarkdown` to keep direct children that satisfy
  `isDisplayMathElement(child)` even when the tag is `span`.
- Regenerated `tools/chatgpt-dom-export.bookmarklet.js`; installed browser
  bookmarks must be updated again.

## Display math double-dollar nesting fix
- User confirmed display math now appears, but Context Forge rendered literal
  strings like `$ c_n=(-1)^n a_n $` inside a display block.
- Root cause: display math was already serialized as `$$...$$`, then
  `promoteStandaloneMath()` treated that existing display block as a standalone
  single-dollar inline math expression and wrapped it again, producing
  `$$ $...$ $$`.
- Updated `promoteStandaloneMath()` to leave existing `$$...$$` blocks alone.
- Added explicit math handling at the start of `elementToMarkdown()` so math
  elements do not unnecessarily pass through the generic inline serializer.
- Regenerated `tools/chatgpt-dom-export.bookmarklet.js`; installed browser
  bookmarks must be updated again.
